# Avito Analytics Pro

Профессиональное HTML-приложение для аналитики рекламы и продаж Авито на основе Excel-файлов статистики (`.xlsx`).
Полностью работает в браузере — без сервера, без установки, без отправки данных куда-либо.

## Возможности

- 📤 **Загрузка Excel** — Drag&Drop или кнопка, поддержка `.xlsx`/`.xls`, автоматическое определение колонок Авито.
- 📊 **Аналитика и KPI** — показы, просмотры, контакты, расходы, доходы, прибыль, CTR, CPC, CPM, CPL, CPA, ROI, ROMI, маржинальность, конверсии.
- 📈 **Графики** — динамика по дням, эффективность объявлений, распределение расходов, воронка продаж, тепловая карта активности.
- 🔍 **Фильтры** — по периоду, категории, региону, типу рекламы, бюджету, CTR, прибыльности.
- 🏆 **Рейтинг объявлений** — лучшие/худшие с подсветкой убыточных.
- 💡 **AI-рекомендации** по оптимизации рекламы.
- 🔮 **Прогноз** контактов на 7 дней вперёд.
- 🆚 **Сравнение периодов** A vs B со всеми ключевыми метриками.
- 💾 **LocalStorage + IndexedDB** — автосохранение датасетов и история загрузок.
- 📤 **Экспорт** — PDF, Excel, CSV, PNG графиков, JSON.
- 🌙 **Тёмная/светлая темы**, glassmorphism, неоновые акценты, адаптивный дизайн.

## Запуск

Просто откройте `index.html` в современном браузере (Chrome, Firefox, Edge, Safari).

```bash
# опционально — локальный сервер для удобства
python3 -m http.server 8080
# → http://localhost:8080
```

## Структура

```
.
├── index.html       — главный HTML, разметка SaaS-уровня
├── css/styles.css   — стили (glassmorphism, неон, тёмная/светлая темы, адаптив)
└── js/
    ├── parser.js    — чтение Excel и нормализация (SheetJS)
    ├── analytics.js — расчёт KPI / конверсий / групп / прогноза
    ├── charts.js    — графики (Chart.js + DOM heatmap / funnel)
    ├── ui.js        — toasts, модалки, таблицы с сортировкой
    ├── storage.js   — IndexedDB (датасеты) + LocalStorage (настройки)
    ├── export.js    — PDF / Excel / CSV / PNG / JSON экспорт
    └── app.js       — главный контроллер: связывает всё вместе
```

## Какие колонки Excel распознаются автоматически

Парсер ищет колонки по словарю синонимов (русские/английские, регистронезависимо):

| Поле        | Распознаваемые названия колонок |
|-------------|--------------------------------|
| Дата        | дата, date, день, период       |
| Объявление  | название, заголовок, объявление, title |
| Категория   | категория, раздел, тип         |
| Регион      | регион, город, локация, region, city |
| Показы      | показы, impressions, shows     |
| Просмотры   | просмотры, views               |
| Контакты    | контакты, обращения, leads     |
| Сообщения   | сообщения, messages, чаты      |
| Звонки      | звонки, calls                  |
| Избранное   | избранное, favorites           |
| Расходы     | расходы, бюджет, spend, cost   |
| Доход       | доход, выручка, revenue, продажи |

Если колонка не распозналась автоматически — в разделе **Настройки** её можно указать вручную.

## Технологии

- Vanilla JS (ES6+)
- [Tailwind CSS](https://tailwindcss.com) (через CDN)
- [Chart.js](https://www.chartjs.org/)
- [SheetJS (xlsx)](https://sheetjs.com/)
- [jsPDF](https://github.com/parallax/jsPDF) + autotable
- [html2canvas](https://html2canvas.hertzen.com/)

Все данные хранятся локально (LocalStorage + IndexedDB) и никогда не покидают браузер.
